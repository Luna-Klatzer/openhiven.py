    Source Code 路 Docs 路 on_init()

    The user logged successfully into the account, and the data will now be sent back to initialise the client-side.

    Expected json-data

"d": {
    "user": {
        "username": str,
        "flags": str | int | None,
        "name": str,
        "id": str,
        "icon": str | None,
        "header": str | None,
        "presence": str | None
    },
    "settings": {
        "user_id": str,
        "theme": None,
        "room_overrides": {
            // room preferences mapped to their idz

                "notification_preference": int 
            }
        },
        "onboarded": unknown,
        "enable_desktop_notifications": unknown
    },
    "relationships": {
        "id": {
            "user_id": str,
            "user": {
                "username": str,
                "flags": str | int | None,
                "name": str,
                "id": str,
                "icon": str | None,
                "header": str | None,
                "presence": str | None
            },
            "type": int,
            "last_updated_at": str | None
        },
    },
    "read_state": {
        "id": {
          "message_id": str,
          "mention_count": int
        },
    },
    "private_rooms": [{
        "default_permission_override": None,
        "description": str,
        "emoji": object | None,
        "house_id": None,
        "id": str,
        "last_message_id": str | None,
        "name": str,
        "owner_id": str,
        "permission_overrides": None,
        "position": None,
        "recipients": [{  
            // User Object
            "username": str,
            "flags": str | int | None,
            "name": str,
            "id": str,
            "icon": str | None,
            "header": str | None,
            "presence": str | None
        } ... ],
        "type": int
    } ... ],
    "presences": {
        "id": {
          "username": str,
          "flags": str | int | None,
          "name": str,
          "id": str,
          "icon": str | None,
          "header": str | None,
          "presence": str | None
        }
    },
    "house_memberships": [{
        "id": {
            "user_id": str,
            "user": {
                // User Object
                "username": str,
                "flags": str | int | None,
                "name": str,

    HOUSE_DOWN¶

    Source Code 路 Docs on_house_down / on_house_delete 路 on_house_down() 路 on_house_delete()

    HOUSE_DOWN represents two types of events:

    • House Deletion - A House was entirely deleted.
    • House Downtime - The data of a house failed to load, and the server has issues recovering it.

    If the given variable unavailable is True, the house with that id is currently down but not deleted.

    Expected json-data
    "op": 0,
"d": {
    "unavailable": bool,
    "house_id": str
}
